Contributing to CHB project
===========================

Thank you very much for your interest in contributing to the CHB
project. Any contribution is welcome and feel free to submit a PRs or an
issue on the repository's main page.

**NOTE:** This document assumes you have already read the project
documentation and are familiar with the CHB-Bootloader code structure.
It is recommended to review all project documentation to get an idea of
how it works and to have performed tests yourself.

Also check the **TODO** file to get an idea of the project roadmap.

A good way to contribute to the project is by viewing the roadmap or
**TODO** list to familiarize yourself with the project.

Project scope and rules
-----------------------

Contribution rules:

- All new features must be consistent with the purpose of the bootloader
  and existing utilities.

- Experimental features should be clearly marked in the code and in the
  PR.

- Clear documentation and usage examples are encouraged.

Reporting issues
----------------

To report bugs or suggest improvements:

- Open an **issue** in the repository:

    https://github.com/Andres2626/CHB-Bootloader/issues

- Provide clear steps to reproduce the problem

- Include full error messages and your operating system if relevant

Commit standards
----------------

Check https://www.conventionalcommits.org/en/v1.0.0/ for more info

Submitting a PR
---------------

1. Fork the repository and create a new branch:

.. code-block:: bash 

    git checkout -b feature/my-feature

2. Make your changes and commit them clearly

3. Push your branch to the remote repository:

.. code-block:: bash 
    
    git push origin feature/my-feature

4. Open a PR against the *main* branch

5. Wait for review and respond to feedback
